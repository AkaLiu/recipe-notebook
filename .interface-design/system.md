# Recipe Notebook System

## Direction

- 产品定位：家庭菜谱册，不是营销站，也不是博客模板。
- 感受目标：温暖、安静、可阅读，像一本经常翻阅的厨房手账。
- 用户任务：按菜系快速找菜，查看配料用量与步骤，收藏常做菜，并导出整本 PDF 菜谱。
- Signature：食材纸条式信息带。每道菜用轻量标签直接展示季节、步骤数、配料数、时长等厨房信息。

## Color System

- 页面底色：纸张米白 `#f7f1e8`
- 深一层纸色：`#efe5d6`
- 卡片底色：`rgba(255, 251, 245, 0.82)`
- 主文字：`#2d241f`
- 次级文字：`#6c6058`
- 弱文字：`#9d8f85`
- 主边框：`rgba(82, 64, 51, 0.18)`
- 强边框：`rgba(82, 64, 51, 0.34)`
- 主题强调色：陶土红 `#b85b45`
- 辅助强调色：鼠尾草绿 `#7f9470`
- 其他辅助：酱汁棕 `#c28a57`，莓果红 `#8f3d4e`

## Typography

- 标题字体：`Newsreader`
- 正文字体：`Source Sans 3`
- 数据/标签字体：`IBM Plex Mono`
- 标题使用紧凑负字距，正文保持高可读行高。

## Spacing

- 基础单位：`8px`
- 常用节奏：`16 / 24 / 32 / 48 / 64`
- 大区块优先留白，卡片内部保持对称 padding。

## Depth Strategy

- 采用“轻边框 + 轻底色差 + 极弱阴影”。
- 不使用重阴影、不使用强烈玻璃感、不使用高对比硬分层。
- 层级分为：`paper -> card -> pin/tag`

## Layout Patterns

- 顶部固定导航，透明纸张感背景，使用明确 z-index。
- 首页 Hero 左文右概览卡。
- 菜谱区采用“左侧 spotlight + 右侧卡片网格”。
- 详情区用双栏结构：左侧配料，右侧步骤。
- liked 区域放在首页靠前位置，作为用户的常做菜书签层。

## Component Patterns

### Recipe Card

- 顶部必须有配图。
- 图片下叠加信息纸条，展示季节、步骤数、配料数等。
- 右上角固定 like 按钮。
- 卡片正文展示菜系、名称、简介、时长、分量、难度。

### Spotlight

- 当前选中菜谱的大卡片。
- 展示完整主信息，并提供跳转到详情区的动作。

### Recipe Sheet

- 标题区包含名称、简介、配图、meta 和 like。
- 内容区固定拆成“配料与用量” / “制作过程”。
- 底部单独保留“厨房提示”。

### Liked Shelf

- 只展示已 like 的菜。
- 没有 liked 数据时显示空状态，不要留白。

### PDF Export

- 导出文档必须包含封面、目录、全部菜谱详情。
- 风格与网页一致：纸张底色、编辑感标题、单色信息标签。
- 打印页按 A4 组织，不做炫技动画。

## Interaction Rules

- 所有可点击元素都要有 `cursor: pointer`。
- hover 只允许颜色、边框、轻微位移变化，避免布局抖动。
- like 状态必须可视化区分。
- 尊重 `prefers-reduced-motion`。

## Avoid

- 冷白 SaaS 面板感
- 深色监控页风格
- 多重强调色同时竞争
- 厚重投影
- 装饰性无意义渐变
- emoji 图标
